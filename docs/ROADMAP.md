# 🗺️ Roadmap do Crystal Legacy

**Versão:** 0.1 Alpha

Este documento descreve a evolução planejada do Crystal Legacy. O roadmap pode mudar ao longo do projeto, mas sempre respeitando a Constituição.

---

## Fase 1 — Fundação

**Objetivo:** criar a base do projeto.

- [x] Definir missão, visão e valores.
- [x] Criar a Constituição.
- [x] Criar a estrutura de pastas.
- [x] Criar repositório no GitHub.
- [x] Configurar Git local.
- [x] Primeiro push.
- [ ] Adicionar documentação inicial ao repositório.
- [ ] Revisar README principal.

---

## Fase 2 — Ambiente de Desenvolvimento

**Objetivo:** preparar um ambiente organizado para desenvolver o Crystal Legacy.

- [ ] VS Code configurado.
- [ ] Git funcionando com fluxo claro.
- [ ] GitHub conectado.
- [ ] Extensões essenciais instaladas.
- [ ] Padrão de organização do projeto definido.

---

## Fase 3 — Boot

**Objetivo:** personalizar a experiência inicial do sistema com segurança.

- [ ] Documentar o fluxo de boot.
- [ ] Corrigir entrada do Windows no GRUB.
- [ ] Criar pasta do tema GRUB.
- [ ] Criar `theme.txt`.
- [ ] Definir fundo, fontes, cores e menu.
- [ ] Testar alterações sem comprometer o boot.

---

## Fase 4 — Plymouth

**Objetivo:** criar a tela de carregamento do sistema.

- [ ] Estudar estrutura de temas Plymouth.
- [ ] Criar animação visual leve.
- [ ] Testar desempenho.
- [ ] Documentar instalação e rollback.

---

## Fase 5 — Login

**Objetivo:** personalizar o SDDM.

- [ ] Estudar estrutura de temas SDDM.
- [ ] Criar tela de login com identidade Crystal Legacy.
- [ ] Manter clareza e facilidade de uso.

---

## Fase 6 — Desktop

**Objetivo:** criar a identidade visual do KDE Plasma.

- [ ] Definir wallpaper oficial.
- [ ] Criar tema visual.
- [ ] Organizar painel, atalhos e fluxo de uso.
- [ ] Documentar configurações.

---

## Fase 7 — Ferramentas Próprias

**Objetivo:** desenvolver aplicações próprias do Crystal Legacy.

- [ ] Crystal Hub.
- [ ] Crystal Backup.
- [ ] Crystal Settings.
- [ ] Ajuda contextual.

---

## Fase 8 — Cavaleiro

**Objetivo:** desenvolver o assistente opcional do Crystal Legacy.

- [ ] Definir arquitetura.
- [ ] Criar protótipo em texto.
- [ ] Avaliar suporte a voz.
- [ ] Garantir funcionamento opcional e não invasivo.

---

## Fase 9 — Alpha

**Objetivo:** consolidar a primeira experiência completa do Crystal Legacy.

- [ ] Documentar instalação.
- [ ] Testar estabilidade.
- [ ] Criar checklist de versão.
- [ ] Preparar primeira versão pública ou privada.

---

## Regra do Roadmap

Nenhuma fase deve avançar sem que a anterior esteja minimamente estável.

> **Planejar → Construir → Testar → Documentar → Evoluir.**
